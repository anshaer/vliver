歡迎使用 **ASE (Animation Project)** 開源網頁動態組件系統。本系統集結了現行的原生 Web 標準技術（HTML5 Canvas, WebP, WebM），**完全不需要支付任何平台訂閱費或軟體授權費**。本手冊將引導您如何在桌機或**平板載具**上，快速完成從素材輕量化、視覺化補間動畫拉線、事件埋藏，到最終一般網頁嵌入的完整開發閉環。

一、 素材輕量化減肥篇 (compressor.html)

若直接將未經壓縮的超高解析度繪圖原檔或螢幕截圖直接塞入動畫中，會造成最終導出的專案包過於肥大，導致使用者網頁載入卡頓。在開始製作起動態前，必須先使用優化工具進行物理輕量化。

操作步驟：

  1. 雙擊開啟 compressor.html 網頁工具。
  2. 根據您要製作的動態組件性質，點擊最上方由研究團隊推導的「建議尺寸黃金面板」（例如：若為四肢元件點選 450x450，若為頭像點選 200x200）。系統會自動將最大寬高上限灌入參數欄中。
  3. 將您的原始大圖（PNG/JPG）拖曳或選取上傳至上傳區。
  4. 導出格式推薦保持預設的 WebP 格式，其具備極高的像素壓縮比且完美保留 8-bit 透明通道。
  5. 拉動質量滑桿（預設 0.80），右側會即時換算並顯示縮放後的點陣圖，以及成功瘦身的百分比（平均可精簡 90% 以上檔案大小）。
  6. 點擊下方藍色按鈕 「下載優化後的輕量化圖片」，留存作為動畫素材。

二、 多圖層補間動畫製作篇 (editor.html)

本編輯器支援傳統滑鼠操作，同時全面支援**智慧型平板電腦（如 iPad、Android 平板）的觸控手勢與觸控筆**。您可以直接在平板上完成複雜的補間拉線與時間軸調校。

操作步驟：

  1. 開啟可視化動態製作組件 editor.html。
  2. 在頂部功能列設定您網頁舞台的「畫布寬度」、「高度」以及本專案的「總時間節點數（預設 50 影格）」。
  3. 在左側素材匯入區，選取剛剛在第一步中利用 WebP 壓縮好的所有輕量化素材。圖片載入後會自動化為獨立圖層堆疊至畫布中央。
  4. 圖層與錨點排序： 您可以在左側圖層管理欄點擊「▲」或「▼」調整前後圖層遮擋關係。在右側面板中，可微調 Pivot X / Y 來改變圖層旋轉或縮放時的核心錨點位置（系統預設 0.5 為正中央，會以紅色原點把手在舞台呈現）。
  5. 編輯影格： 拉動底部時間軸滑桿（或輸入精準數字）切換影格節點。在畫布上，您可以直接用**滑鼠拖曳（或在平板上用手指/觸控筆直接點擊拖放）**圖片進行位移、調整右下角手把進行等比例縮放、或是拉動上方手把進行矩陣旋轉。
  6. 高階緩動曲線： 當您在非 0 影格改變圖片狀態時，系統會異步活化該格關鍵影格。您可在右側下拉選單為該區段配置 Bounce (物理回彈) 或 EaseInOut (雙向漸變) 等高階數學插值曲線，賦予組件極佳的打擊彈性。
  7. 埋藏事件標記： 若想讓動畫在特定影格觸發網頁行為，可在「時間軸事件標記觸發器」欄位輸入自訂字串標記（例如：play_vfx）。
  8. 點擊底部「播放預覽」確認動態完全流暢。
  9. 專案匯出： 確認無誤後，點擊頂部功能列的 **「匯出專案檔 (.ase)」** 下載二進位封包。
    💡 軟體工程安全防禦小知識：
    本生態系導出的 .ase 專案檔本質上為高效標準 ZIP 容器。為防範使用者誤將日常生活中不相關的壓縮檔（如相片壓縮包）丟入系統引發程式碼崩潰，工具鏈在前端實施了副檔名沙盒過濾。請維持 .ase 命名，以維護多圖片組的完整性與系統健全度。

三、 專案快速檢視與除錯篇 (檢視ASE.html)

當您導出多個 .ase 專案，或是想要快速驗證特定時間軸事件是否精準撞擊觸發時，不需繁瑣地修改應用網頁程式碼，可直接利用專門的黑盒子檢視器進行除錯。

操作步驟：

  1. 開啟專案快速檢視工具 檢視ASE.html。
  2. 直接將您剛導出的 animation_project.ase 檔案拖曳進網頁中央的 dashed 虛線區域（或選取上傳）。
  3. 系統解壓核心會立刻調用 API 啟動動畫。如果您的動畫帶有完全透明背景，可以點擊 **「切換棋盤透明網格底色」** 按鈕，方便裸眼觀察角色邊緣去背細節。
  4. 下方的 **「時間軸事件標記即時日誌監控」** 黑色面板會啟動全域監聽。一旦播放線行經您當初在編輯器埋設事件的影格，面板會立刻精準印出觸發日誌（例如：[時間] 節點 25 💥 觸發標記事件 → "hit_target"），方便開發者驗證互動邏輯。
圖 3. 檢視ASE.html 內建棋盤底色切換與事件監控黑盒子之快速檢視工具

四、 一般網頁外部 API 嵌入實戰篇 (hello2.html)

本研究開發之播放核心 API（ase-player.js）已成功部署於具有邊緣加速、位於台灣彰化的 Google Cloud Storage (GCS) 頂級高速資料中心，並配置好全域 CORS 跨來源開放規則。任何第三方的一般網站或部落格，**皆不需下載本地外連 JS 檔**，只需遵循以下三個步驟，即可零時延引用網頁組件。實作樣貌可參考驗證網頁：hello2.html。

通用嵌入代碼範例：

請在您要調用動畫的網頁 HTML 檔案中，參考並貼入以下標準結構：

<!DOCTYPE html>
<html lang="zh-TW">
<head>
    <meta charset="UTF-8">
    <title>一般網頁嵌入動畫展示</title>
